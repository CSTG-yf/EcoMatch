## 3. 现有能力与改造边界

### 3.1 可直接复用

- Chat BI 查询、解析和执行主流程。
- 规则与 LLM 组合的 NL2SQL 解析机制。
- Schema Mapper、Semantic Parser、Semantic Corrector 和 SQL Translator 扩展接口。
- 指标、维度、数据集、业务术语和知识库管理。
- MySQL、PostgreSQL、ClickHouse、StarRocks、Presto、Trino、DuckDB 等数据库适配。
- 模型可见权限、敏感列拦截和行级过滤。
- 会话历史、问答反馈、移动端入口、图表展示、下钻、CSV 和图片导出。
- SQL 精确匹配、执行匹配和批量问数评测脚本。

### 3.2 必须改造

- 银行业术语、简称、口径、指标和场景数据尚未形成可交付数据包。
- 当前多轮改写只使用最近一轮历史，无法证明满足连续 10 轮要求。
- 跨数据库能力以多数据源适配为主，缺少单次查询的联邦执行闭环。
- 图表推荐主要是前端启发式规则，缺少标注数据、推荐服务和准确率评测。
- 数据解释默认关闭，缺少口径引用、证据约束、异常归因和幻觉校验。
- 缺少动态脱敏、操作审计、异常行为告警和导出安全控制。
- 缺少自定义分析看板、受控分享、XLSX/PDF 报表导出。
- 缺少国产数据库、国产操作系统、ARM64 环境的兼容性证明。
- 当前 benchmark 以单轮为主，缺少多轮、权限、图表和业务理解度评测。

## 10. 三条核心演示链路

### 场景 A：经营分析

问题示例：“本季度各农商行存贷款余额、同比增速和计划完成率如何？下钻到增速最低的机构。”

必须展示：多指标查询、同比计算、机构下钻、口径来源、趋势图、业务结论和导出。

### 场景 B：风险管控

问题示例：“近三个月逾期率持续上升的机构和产品有哪些？查看高风险客户分布。”

必须展示：异常识别、权限差异、客户字段脱敏、风险解释、越权阻断、审计和告警。

### 场景 C：客户营销

问题示例：“筛选近半年存款增长但未购买理财的高价值客户，按机构统计并生成营销分析。”

必须展示：跨源查询、客群筛选、图表推荐、业务解释、受控分享和访问留痕。

## 11. 发布门禁

以下任一条件未满足，不进入比赛交付版本：

- 意图识别准确率低于 94%。
- 复杂 SQL 执行成功率低于 90%。
- 平均响应时间高于 3 秒，或大量请求依赖超时重试完成。
- 10 轮连续对话存在上下文串扰或关键条件丢失。
- 三类业务场景或 300 项指标未完成。
- 图表匹配准确率或业务人员理解度低于 90%。
- 存在查询、历史、导出、分享或模型调用路径的越权和敏感数据泄露。
- 审计链路不能还原查询、权限、脱敏、导出和分享行为。
- 国产化环境无法完成部署、启动、查询和回滚。

## 12. 项目风险与应对

| 风险 | 影响 | 应对措施 |
| --- | --- | --- |
| 银行数据和指标到位较晚 | 无法优化和验证准确率 | 第 1 周冻结数据模板，先用同构脱敏样例开发 |
| 3 秒指标口径不明确 | 验收争议 | 明确是否包含 LLM 和数据库执行，同时记录端到端和分阶段耗时 |
| 300 项指标集中导入导致口径冲突 | 查询结果不一致 | 建立版本、审批、冲突检测和冻结测试集 |
| 跨库查询性能不可控 | 无法满足响应目标 | 使用联邦引擎、资源组、预聚合和缓存，限制明细结果规模 |
| 大模型产生错误业务结论 | 降低可信度 | 解释必须引用结果数据和指标口径，并执行数值一致性校验 |
| 安全能力只覆盖页面 | 可被接口绕过 | 所有权限和脱敏策略在后端统一执行，前端仅负责展示 |
| 国产数据库方言差异 | SQL 无法执行 | 提前建立方言测试矩阵和目标数据库集成环境 |

## 13. 完成定义

每个任务只有同时满足以下条件才可标记完成：

- 代码通过格式检查、单元测试和模块集成测试。
- 接口具备清晰的输入、输出、错误码和权限说明。
- 有可重复执行的验收数据或测试脚本。
- 安全相关任务包含正向授权和反向越权测试。
- 前端任务完成 PC 与移动端检查，无明显溢出、遮挡和不可操作状态。
- 核心指标被统一评测平台记录，可与上一版本比较。
- 变更附带配置、部署、回滚和故障排查说明。
