# FE-07 指标与术语治理后台实现说明

## 实现范围

- 新增 `/governance` 指标治理菜单和独立工作台，支持按语义模型切换治理范围。
- 指标治理列表展示状态、版本、责任部门、来源系统、业务口径和生效区间。
- 指标详情抽屉支持治理信息维护、版本创建和比较、提交审批、通过/驳回、发布、回滚和停用。
- 血缘页展示上游指标、下游指标、关联维度、引用数据集和机构映射。
- 口径冲突页展示冲突等级、类型、冲突键和涉及指标，并可直接进入指标处理。
- 术语页支持按名称、同义词和说明检索当前业务域术语。
- 批量导入向导支持 Excel 选择、只读校验、错误定位、目标字段配置、指标与术语导入及结果统计。
- 导入成功后自动初始化新增指标的治理档案，形成导入、校验、治理和发布闭环。

## 接口对接

- BE-01：`/api/semantic/bank/resources/validate`、`/api/semantic/bank/resources/import`。
- BE-02：`/api/semantic/metric-governance/**` 指标档案、版本、审批、发布、冲突和血缘接口。
- 现有语义接口：业务域、模型、指标和术语查询。

## 验收

- 治理模块定向 TypeScript 5.9 检查通过。
- `pnpm build:os` 生产构建通过，生成 `/governance` 页面和独立异步资源。
- 使用真实浏览器和接口桩完成 1440px 桌面及 390px 窄屏检查，页面无内容遮挡；表格在窄屏内独立横向滚动。
- 本地开发服务运行于 `http://localhost:9000/webapp/governance`，后端接口代理目标为 `http://127.0.0.1:9080`。

说明：仓库默认 TypeScript 4.9 无法解析当前 Pro Form 依赖中的 TypeScript 5 类型语法，因此定向类型检查使用依赖树中的 TypeScript 5.9；Umi 生产构建已完整通过。
